This is a second article in a series on writing Opscode Chef cookbooks the Berkshelf Way. Here’s a link to Part 1.

In this article, part 2, we’re going to create a new database recipe. In Part1 MyFace is just a web application serving up a static page. In Part 2, we’re going to enhance MyFace so that it stores account information in a persistent database. We’re going to use MySQL for the database.

Iteration #8 - Create user table

We’ve installed MySQL, but we don’t have a database table yet. Now we’re going to create a database table to store information about our users with another cookbook, the database cookbook.

Add the database cookbook as a dependency in the metadata.rb file:

Berkshelf automatically populates MySQL passwords for you. They were configured in the Vagrantfile when you ran berks cookbook in Part 1:

...
config.vm.provision :chef_solo do |chef|
  chef.json = {
    :mysql => {
      :server_root_password => 'rootpass',
      :server_debian_password => 'debpass',
      :server_repl_password => "replpass'
    }
  }
  ...
end
...

You can refer to these passwords in your Chef recipes, which we will do when we describe the database.
